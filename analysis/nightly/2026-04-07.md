## Key Observations

1. **No graph_break regressions.** Zero models went from full_graph to graph_break. The nightly is strictly better than v2.10 for graph break coverage.

2. **3 real fixes landed:** FalconMambaModel, MambaModel, MraModel all moved from graph_break to full_graph. These represent genuine Dynamo improvements.

3. **6 infra fixes:** Worker errors from v2.10 (likely harness/environment issues) are resolved in the nightly environment.

4. **3 "regressions" are transformers-side, not PyTorch:** BambaModel, GraniteMoeHybridModel, and JambaModel moved from graph_break to eager_error due to Cache API changes in transformers 5.5.0 (HybridCache constructor changes). These models were already broken (graph_break); the error mode changed.

5. **Dynamic shapes delta is small:** Only 3 additional eval graph breaks with dynamic=true vs static, and 10 more with dynamic=mark. Dynamic shape support is mature for this corpus.
